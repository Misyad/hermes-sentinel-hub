## 8. ACCESSIBILITY NOTES

**✅ Passed:**
- All interactive elements keyboard accessible (Tab, Enter, Space)
- Search input has placeholder text
- Filter buttons keyboard navigable
- Action buttons have `title` attributes for tooltips
- Sortable headers keyboard accessible (click/Enter)
- Semantic HTML: `<button>`, `<input>`, `<table>`
- Status not indicated by color alone (severity/status badges have text)
- Sufficient contrast (7:1+):
  - White text on zinc-950: 18:1
  - Emerald on zinc-950: 8.2:1
  - Red/Amber badges on dark: 7:1+

**⚠️ Improvements Needed:**
- Filter buttons should have `role="radiogroup"` and individual buttons `role="radio"` + `aria-checked`
- Search input should have `aria-label="Search alerts"`
- Empty state icon should have `aria-hidden="true"`
- Table should have `role="table"` (implicit via semantic HTML, but explicit is better)
- Action buttons should have `aria-label` not just `title` (for screen readers)

**❌ Failures:**
- None blocking

**Recommendation:** Add ARIA labels and roles in next iteration
